# Global Agent Rules

## 最高约束

1. 本项目是全新项目，禁止任何兼容性设计。
2. 数据模型只服务当前确认需求，不为历史版本、旧前端、旧数据库做适配。
3. Python 项目是业务行为参考，不是 Go 项目的技术架构模板。
4. shared-docs 是跨端事实源；有疑问先读文档，不要猜。

## 项目关系

- Go 后端：`house-manager`，承载 auth、主数据、发房、后台等核心后端能力。
- Python 后端：`ai_house`，保留 AI chat 相关能力和迁移参考。
- 小程序：C 端用户找房入口。
- 发房系统：房源录入和发布入口。
- 后台管理：运营、审核、数据维护入口。

## 工作方式

1. 先确认当前任务属于哪个端或哪个模块。
2. 先读对应 Agent 文档，再读对应 API / schema / flow。
3. 修改接口时必须同步 API 文档。
4. 修改数据库字段、枚举、索引时必须同步 schema 文档。
5. 修改跨模块流程时必须同步 flows 或 changes。
6. 不要把历史输入文档当成当前事实源，当前事实源以规范类文档和最新 ADR 为准。
