# AGENTS

## 作用范围

本规则适用于 `shared-docs/` 目录下全部文件与子目录。

## 仓库定位

这是共享契约仓库，不承载实现代码。

## 契约优先级

1. API 路径、方法、请求响应：以 `api/openapi.yaml` 为准（唯一 API 契约源）。
2. 字段语义与枚举：以 `schemas/` 为准，且必须与 OpenAPI 保持同步。
3. 页面到接口调用流程：以 `contracts/` 为准。

## 编辑规则

1. 输入类文档（需求、会议结论、阶段性确认）默认作为历史记录保留，不直接改写历史语义。
2. 规范类文档（API/Schema）是当前事实来源；接口变更必须同时更新：
   - `api/openapi.yaml`
   - `schemas/` 下对应对象文档
3. 若“当前实现/最终策略”与输入类文档存在差异：
   - 在 `contracts/` 下新增日期化说明文档（例如：`*-adjustments-YYYY-MM-DD.md`）。
   - 通过“新增说明”表达差异，不覆盖原始输入记录。
4. 文档尽量追加式更新，避免覆盖历史上下文。
5. 发生跨文件调整时，确保契约一致，避免“OpenAPI 与说明文档不一致”。

## 协作约定（前后端共享）

1. 本规范同时面向后端与前端.
2. 讨论接口是否可用、字段是否变更时，以 `api/openapi.yaml` 为统一判断依据。
3. 讨论“为什么和原需求不一致”时，以 `contracts/*-adjustments-YYYY-MM-DD.md` 为统一解释依据。
4. 新增接口前，先补契约再联调；变更接口后，先更新契约再通知对端。

## 交付检查清单

- [ ] `api/openapi.yaml` 已同步接口变更
- [ ] `schemas/` 已同步字段语义、枚举与约束
- [ ] 若存在口径调整，已新增一份 `contracts/*-adjustments-YYYY-MM-DD.md`
- [ ] 输入类文档与规范类文档语义一致，或已有差异说明文档
