# ADR 0001: 统一响应与错误格式

## 背景

当前前后端联调需要稳定、低歧义的统一返回结构，避免每个接口独立定义成功/失败格式。

## 决策

- 成功响应统一使用 `{ code, message, data }`。
- 业务成功约定 `code = 0`。
- 失败时在 HTTP 层表达状态码，错误细节优先使用 `detail`。
- 当前实现中，`detail` 多数仍为字符串；结构化错误对象保留为后续收口目标。

## 原因

- 统一包裹可降低前端拦截器与页面层分支复杂度。
- `code` 适合程序分支；`message` 适合展示与日志；`details` 适合扩展字段级错误。
- 与当前后端实际返回结构一致。

## 收益

- 跨接口处理方式一致。
- 联调成本下降。
- 错误监控与排查更清晰。
