# Admin

后台管理系统是平台员工工作台，用于审核、运营、分发、跟进和账号权限管理。

## 产品定位

三端分工：

- `miniapp`：租客找房、聊天、收藏、预约
- `publish`：房东录入和维护自己的房源
- `admin`：平台员工管理平台流程

后台管理不是发房系统增强版，也不是房东视角页面集合。

## 第一阶段范围

第一阶段只做平台最小闭环：

1. 员工登录
2. 员工 / 角色 / 权限管理
3. 发房方管理
4. 房源管理总览
5. 房源审核

## 第二阶段候选

- 预约看房
- 客户消息
- 营销管理

## 边界

- 后台管理和发房系统是不同前端。
- 后台管理 API 独立设计，不直接等于发房系统 API。
- 后台管理可以复用底层 Go 后端能力，但不复用 miniapp / publish 的端侧 service。
