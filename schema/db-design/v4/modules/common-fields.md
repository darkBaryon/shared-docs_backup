# 公共字段（V4 全集合通用）

适用范围：V4 下所有 Mongo 集合。

## 1. 公共字段定义

| 字段 | 类型 | 必填 | 默认值 | 字段含义 |
| --- | --- | --- | --- | --- |
| `_id` | objectId | 是 | 无 | Mongo 文档主键，由 Mongo 自动生成 |
| `created_at` | int64 | 是 | 当前时间秒 | 文档创建时间（Unix 秒） |
| `updated_at` | int64 | 是 | 当前时间秒 | 文档最后更新时间（Unix 秒） |
| `status` | int | 是 | `1` | 文档状态字段，用于区分有效/禁用/删除等状态 |
| `version` | int | 是 | `1` | 文档版本号，用于并发更新控制和结构演进 |

## 2. status 枚举约定

| 值 | 含义 | 说明 |
| --- | --- | --- |
| `0` | 未指定 | 默认保留值，不作为正常业务态使用 |
| `1` | 有效 | 当前记录正常可用 |
| `-1` | 删除/禁用 | 软删除或禁用态，默认不对业务端展示 |

说明：
- 若某模块需要更多业务状态（如审核中、已上架），在该模块文档中扩展定义。
- 扩展状态必须遵循项目规范：`1..99` 为有效业务态，`-1..-99` 为无效/删除态。

## 3. 通用行为约定

| 字段 | 写入规则 |
| --- | --- |
| `created_at` | 仅创建时写入，后续不改 |
| `updated_at` | 每次更新都必须刷新 |
| `status` | 原则上使用软删除，不直接物理删除 |
| `version` | 每次更新 +1（用于乐观锁可选实现） |
